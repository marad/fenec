## Tech Debt

**CONF-04 active-provider hot-reload gap:** main.go reload callback calls providerRegistry.Update() but does not propagate the new provider instance to r.provider in the REPL. If user changes API key or URL in config.toml while chatting, the active session does not auto-pick it up. Workaround: /model <provider>/<model> to fetch fresh instance.

**Phase 9 go.mod:** BurntSushi/toml and fsnotify listed as // indirect despite being direct imports. Non-functional; go mod tidy resolves.

**Pre-existing test failures (not v1.1 regressions):**
- TestLoadSystemPromptFromFile: macOS os.UserConfigDir() ignores XDG_CONFIG_HOME; test isolation issue only
- 7 internal/tool DeniedPath tests: assert Linux /etc/shadow behavior; fail on macOS
