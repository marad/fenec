# Milestone v1.1 Multi-Provider Support — Audit Report

**Audited:** 2026-04-13
**Phases:** 5/5 complete (Phases 7-11)
**Overall status:** `gaps_found` — 3 partial requirements, 3 integration issues, 1 flow gap

## Executive Summary

Milestone v1.1 successfully delivers multi-provider support. All 15 requirements are functionally implemented, all 5 phases passed their own verification (37/37 must-haves), and all critical flows work end-to-end. However, cross-phase integration checking revealed 3 gaps that prevent a clean audit pass:

1. **OpenAI streaming thinking not delivered** (OAIC-01, OAIC-02) — The OpenAI adapter extracts `<think>` tag content but never calls the `onThinking` callback in its streaming path. Thinking-capable models via LM Studio or cloud providers silently lose their reasoning display in pure-chat mode.

2. **`ProviderConfig.DefaultModel` is a dead field** (CONF-01, ROUT-01) — The TOML field parses correctly but no code path reads it. Users setting `default_model` per-provider see the value silently ignored.

3. **Hot-reload doesn't refresh active REPL session** (CONF-04) — `providerRegistry.Update()` swaps providers correctly, but the REPL holds a stale `r.provider` reference until the user runs `/model` or restarts. URL and API key changes don't take effect mid-session.

## Integration Issues

### 1. OpenAI streaming `onThinking` never invoked

**Location:** `internal/provider/openai/openai.go` `chatStreaming()`
**Issue:** `onThinking` parameter accepted but not called; `<think>` tag extraction happens post-stream but content is silently stripped
**Fix approach:** Parse `<think>` tags incrementally during stream and invoke `onThinking` per chunk, similar to Ollama adapter behavior

### 2. `ProviderConfig.DefaultModel` dead field

**Location:** `internal/config/toml.go:28` (field) and `main.go:147-149` (model selection)
**Issue:** Only top-level `Config.DefaultModel` is read; per-provider `default_model` ignored
**Fix approach:** In `main.go`, when `--model` has no prefix or no model part, fall back to provider's `DefaultModel` field before top-level default

### 3. Hot-reload doesn't refresh active REPL provider

**Location:** `main.go:92-116` (reload callback) and `internal/repl/repl.go` (r.provider cache)
**Issue:** Registry swaps providers but REPL's cached reference persists
**Fix approach:** Reload callback should also refresh active provider in REPL, OR REPL should always look up active provider from registry by name rather than caching
