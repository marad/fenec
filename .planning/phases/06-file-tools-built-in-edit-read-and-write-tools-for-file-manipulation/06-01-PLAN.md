  <action>
    Create internal/tool/pathcheck.go in package tool with two exported functions:

    1. `IsDeniedPath(path string) (bool, error)`:
       - Resolve path to absolute with `filepath.Abs(path)`
       - Resolve symlinks with `filepath.EvalSymlinks(absPath)`. If EvalSymlinks fails (file doesn't exist yet, e.g. write_file creating new file), resolve the parent directory with `filepath.EvalSymlinks(filepath.Dir(absPath))` and append `filepath.Base(absPath)`.
       - If both resolve attempts fail, return `(true, err)` -- deny on resolution failure (fail closed).
       - Compare resolved path against deny list prefixes. The deny list is built in a package-level `init()` function using `os.UserHomeDir()` to expand `~`:
         - `/etc`, `/usr`, `/bin`, `/sbin`, `/boot`
         - `filepath.Join(home, ".ssh")`, `filepath.Join(home, ".gnupg")`
       - Prefix matching: `resolved == prefix || strings.HasPrefix(resolved, prefix + string(filepath.Separator))` -- this prevents /etcetera matching /etc (per D-06, D-07, pitfall 5 from RESEARCH.md).

    2. `IsOutsideCWD(path string) (bool, error)`:
       - Get CWD with `os.Getwd()`
       - Resolve path to absolute with `filepath.Abs(path)`
       - Compute relative path with `filepath.Rel(cwd, absPath)`
       - Return `(true, nil)` if relative path starts with `..`
       - If any resolution fails, return `(true, err)` -- fail closed.

    Imports needed: `os`, `path/filepath`, `strings`.

    Create internal/tool/pathcheck_test.go with individual test functions following the safety_test.go pattern (per-case functions, testify assert). Tests should use `t.TempDir()` for any filesystem tests. For deny-list tests involving home directory, use `os.UserHomeDir()` to construct expected paths.
  </action>
