Rewrite `chatStreaming()` in `internal/provider/openai/openai.go` to parse `<think>` tags incrementally as chunks arrive, routing content to the correct callback.

**State machine approach:**

Add a `thinkParser` struct (unexported, in openai.go) that tracks streaming state:

```go
type thinkParser struct {
    inThink    bool           // currently inside <think> block
    pending    strings.Builder // accumulates partial tag matches
    thinking   strings.Builder // accumulated thinking content
    content    strings.Builder // accumulated non-thinking content
    onToken    func(string)
    onThinking func(string)
}
```

The parser processes each content delta character-by-character (or efficiently with strings.Index):

1. When NOT in think mode:
   - Scan for `<think>` opening tag. Accumulate characters in `pending` if they could be the start of `<think>`. If `pending` reaches `<think>`, switch to `inThink=true` and clear pending.
   - If pending characters turn out not to be `<think>`, flush pending to onToken.
   - Non-pending content goes directly to onToken and content builder.

2. When IN think mode:
   - Scan for `</think>` closing tag. Same pending logic for partial tag match.
   - Content before `</think>` goes to onThinking and thinking builder.
   - When `</think>` found, switch inThink=false.

3. At end of stream, flush any remaining pending buffer appropriately.

**Key implementation detail:** Use `strings.Index` for efficient scanning rather than character-by-character when possible. The common case (no think tags) should be fast — check if the accumulated content string contains `<` at all before engaging the state machine.

**Update chatStreaming() body:**

Replace the current simple loop:
```go
// OLD: just appends to content and calls onToken
for stream.Next() {
    chunk := stream.Current()
    if len(chunk.Choices) > 0 {
        delta := chunk.Choices[0].Delta
        if delta.Content != "" {
            content.WriteString(delta.Content)
            if onToken != nil {
                onToken(delta.Content)
            }
        }
    }
}
```

With:
```go
// NEW: route through thinkParser
tp := &thinkParser{onToken: onToken, onThinking: onThinking}
for stream.Next() {
    chunk := stream.Current()
    if len(chunk.Choices) > 0 {
        delta := chunk.Choices[0].Delta
        if delta.Content != "" {
            tp.process(delta.Content)
        }
    }
}
tp.flush() // flush any remaining pending
```

**Build final message from parser state:**
```go
msg := model.Message{
    Role:     "assistant",
    Content:  tp.content.String(),
    Thinking: strings.TrimSpace(tp.thinking.String()),
}
// No need for extractThinkingFromContent — parser already separated them
```

Remove the `extractThinkingFromContent(&msg)` call at end of chatStreaming (it would be redundant / harmful since content is already separated).

**Preserve extractThinkingFromContent** — it is still used by `chatNonStreaming`. Do not delete it.
