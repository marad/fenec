func (r *Registry) ToolInfo() []ToolInfoEntry {

**create.go** -- per D-01, D-02, D-03, D-04:

Create `internal/tool/create.go` with:

```go
package tool

type CreateLuaTool struct {
    toolsDir string
    registry *Registry
    notifier ToolEventNotifier
}

func NewCreateLuaTool(toolsDir string, registry *Registry, notifier ToolEventNotifier) *CreateLuaTool
func (c *CreateLuaTool) Name() string       // returns "create_lua_tool"
func (c *CreateLuaTool) Definition() api.Tool
func (c *CreateLuaTool) Execute(ctx context.Context, args api.ToolCallFunctionArguments) (string, error)
```

Definition has one required parameter: `code` (string) -- "Complete Lua tool source code. Must return a table with name, description, and execute fields."

Execute flow (per D-02):
1. Extract `code` string from args via `args.Get("code")`. If missing, return Go error `fmt.Errorf("missing required argument: code")`.
2. Write code to temp file via `os.CreateTemp("", "fenec-tool-*.lua")`. Use `defer os.Remove(tmpPath)` for cleanup on all paths.
3. Call `lua.CompileFile(tmpPath)`. On error, return the error as a tool result string (JSON `{"error": "..."}`) -- NOT a Go error. The model needs to see it.
4. Call `lua.NewLuaToolFromProto(proto, tmpPath)`. On error, return as tool result string.
5. Check `c.registry.Has(lt.Name())` -- per D-04, reject if exists: return `{"error": "tool 'X' already exists. Use update_lua_tool to replace it."}`.
6. Also check `c.registry.IsBuiltIn(lt.Name())` -- reject if name collides with a built-in.
7. `os.MkdirAll(c.toolsDir, 0755)` to create directory if needed.
8. Write code to `filepath.Join(c.toolsDir, lt.Name()+".lua")` via `os.WriteFile` (NOT os.Rename -- avoids cross-device issues per research).
9. Re-compile from final path so `scriptPath` on the LuaTool is correct: `proto, _ = lua.CompileFile(finalPath)` then `lt, _ = lua.NewLuaToolFromProto(proto, finalPath)`.
10. `c.registry.RegisterLua(lt)` to register.
11. Call `c.notifier` if non-nil with event="created", name, description.
12. Return success JSON: `{"status":"created","name":"...","description":"...","parameters":["param1","param2"]}` per D-03.

Import `feneclua "github.com/marad/fenec/internal/lua"` using the `feneclua` alias established in main.go. Or use just `lua` import alias -- follow whatever the tool package needs (note: the package is `lua`, but we import it as `feneclua` from outside the lua package. Inside `internal/tool/`, use `"github.com/marad/fenec/internal/lua"` with whatever alias avoids collisions).
