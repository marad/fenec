    NOTE: Use the same package name and gopher-lua import alias established in Plan 01 (either `package lua` with `import glua "github.com/yuin/gopher-lua"` or `package luavm` if Plan 01 went that route). Read the existing files from Plan 01 first to match.

    Types:
    ```go
    // LoadError records a tool that failed to load.
    type LoadError struct {
        Path   string // Absolute path to the .lua file
        Reason string // Human-readable error description
    }

    // Error implements the error interface for LoadError.
    func (e *LoadError) Error() string {
        return fmt.Sprintf("%s: %s", e.Path, e.Reason)
    }

    // LoadResult holds the outcome of scanning a tools directory.
    type LoadResult struct {
        Tools  []*LuaTool   // Successfully loaded tools
        Errors []LoadError  // Tools that failed to load
    }
    ```

    `LoadTools(toolsDir string) (*LoadResult, error)`:
    - Initialize `result := &LoadResult{}`
    - `entries, err := os.ReadDir(toolsDir)` -- if `os.IsNotExist(err)`, return result (empty, no error). If other error, return nil, err.
    - For each entry:
      - Skip if `entry.IsDir()` or not `.lua` suffix: `!strings.HasSuffix(entry.Name(), ".lua")`
      - Build full path: `filepath.Join(toolsDir, entry.Name())`
      - Call `CompileFile(path)`. On error, append `LoadError{Path: path, Reason: err.Error()}`, continue.
      - Call `NewLuaToolFromProto(proto, path)`. On error, append `LoadError{Path: path, Reason: err.Error()}`, continue.
      - Append tool to `result.Tools`
    - Return result, nil

    Insert the following block **after** `registry.Register(shellTool)` and **before** the `// Create and run REPL.` comment:

    ```go
    // Load Lua tools from tools directory.
    toolsDir, err := config.ToolsDir()
    if err != nil {
        slog.Warn("failed to resolve tools directory", "error", err)
    } else {
        result, err := feneclua.LoadTools(toolsDir)
        if err != nil {
            fmt.Fprintln(os.Stderr, render.FormatError(
                fmt.Sprintf("Failed to scan tools directory: %v", err)))
            // Non-fatal: continue without Lua tools.
        } else {
            for _, t := range result.Tools {
                registry.Register(t)
            }
            if len(result.Tools) > 0 {
                slog.Info("loaded Lua tools", "count", len(result.Tools))
            }
            for _, e := range result.Errors {
                fmt.Fprintln(os.Stderr, render.FormatError(
                    fmt.Sprintf("Lua tool load error: %s", e.Error())))
            }
        }
    }
    ```

    Key design decisions:
    - Lua tool loading is NON-FATAL. If ToolsDir fails, log warning and continue. If LoadTools fails, print error and continue. Only individual tool errors are reported; the app still starts.
    - Use `slog.Info` for successful loads (structured logging per stack decisions).
    - Use `render.FormatError` for load errors so they are styled consistently with other errors.
    - Load errors print to stderr so they are visible immediately at startup (per LUA-06).
    - Loaded Lua tools are registered in the same registry as shell_exec, so they appear in system prompt via `registry.Describe()` and are dispatchable via `registry.Dispatch()`.
