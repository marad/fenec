# Milestone v1.3: Profiles & Config

**Status:** ✅ SHIPPED 2026-04-15
**Phases:** 14-19
**Total Plans:** 6

## Overview

Named assistant profiles with custom system prompts, models, and providers — plus config path standardization and conversation reset.

## Phases

### Phase 14: Config Path Migration

**Goal**: Config directory lives at `~/.config/fenec` on all platforms with automatic migration from legacy macOS path
**Depends on**: Phase 13
**Requirements**: CFG-01, CFG-02, CFG-03
**Plans**: 1 plan

Plans:
- [x] 14-01-PLAN.md — ConfigDir standardization, migration logic, and main.go wiring (TDD)

**Key decisions:**
- os.UserHomeDir + hardcoded .config path over os.UserConfigDir — consistent cross-platform behavior

### Phase 15: Clear Command

**Goal**: User can reset conversation mid-session without losing previous context or breaking REPL state
**Depends on**: Phase 14
**Requirements**: CONV-01, CONV-02, CONV-03
**Plans**: 1 plan

Plans:
- [x] 15-01-PLAN.md — /clear command with auto-save, state reset, and test coverage

### Phase 16: Profile Package

**Goal**: Profile data model and file I/O enable creating, loading, and listing named profiles with TOML frontmatter and markdown system prompts
**Depends on**: Phase 14
**Requirements**: PROF-01, PROF-02, PROF-03
**Plans**: 1 plan

**Key decisions:**
- Profile files use +++‑delimited TOML frontmatter (Hugo-style)
- Path traversal protection rejects names with /, \, or .

### Phase 17: System Flag

**Goal**: User can override the system prompt for a single invocation via a file path flag
**Depends on**: Phase 14
**Requirements**: FLAG-01
**Plans**: 1 plan

Plans:
- [x] 17-01-PLAN.md — --system/-s flag with conditional system prompt loading

**Key decisions:**
- Inline file reading in main.go — 5 lines of logic, no abstraction needed
- Hard fail on missing/unreadable --system file — no silent fallback

### Phase 18: Profile Flag

**Goal**: User can activate a named profile at launch, loading both model and system prompt with proper flag precedence
**Depends on**: Phase 16, Phase 17
**Requirements**: FLAG-02, FLAG-03, FLAG-04
**Plans**: 1 plan

Plans:
- [x] 18-01-PLAN.md — --profile/-P flag with model/prompt precedence chains

**Key decisions:**
- pflag.CommandLine.Changed("model") guard for three-layer model/prompt precedence
- Uppercase -P shorthand since -p is taken by --pipe
- Three-layer prompt precedence: --system > profile > config default

### Phase 19: Profile Subcommands

**Goal**: User can manage profiles through dedicated CLI subcommands without interfering with normal REPL operation
**Depends on**: Phase 16
**Requirements**: PROF-04, PROF-05, PROF-06
**Plans**: 1 plan

**Key decisions:**

- Dir-parameter injection pattern enables testing without filesystem config dependency

## Milestone Summary

**Key Decisions:**
- Config path: os.UserHomeDir + hardcoded .config (cross-platform consistency)
- Profile format: +++‑delimited TOML frontmatter + markdown body (Hugo-style)
- Flag precedence: --model > profile > config default; --system > profile prompt > config prompt
- Subcommand routing: pre-pflag os.Args dispatch (avoids Cobra for 3 subcommands)

**Issues Resolved:**
- macOS config path inconsistency (~/Library/Application Support → ~/.config)
- No way to reset conversation mid-session
- No named profiles or system prompt customization

**Issues Deferred:**
- Profile switching mid-session (`/profile` REPL command) — needs conversation context decisions
- `FENEC_CONFIG_DIR` env var override — covers 99% of cases without it
- `XDG_CONFIG_HOME` on Linux — deferred until requested

**Technical Debt Incurred:**
- Phase 15 progress table shows "0/1" plans but was completed (tracking artifact)
- Pre-existing tool package test failures on macOS (not introduced in this milestone)
