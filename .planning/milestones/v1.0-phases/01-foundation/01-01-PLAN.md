       ```go
       package chat

       import (
           "context"
           "fmt"
           "net/http"
           "net/url"

           "github.com/ollama/ollama/api"
       )

       // Client wraps the Ollama API client.
       type Client struct {
           api *api.Client
       }

       // NewClient creates a client connecting to the given host.
       // If host is empty, falls back to OLLAMA_HOST env var, then localhost:11434.
       func NewClient(host string) (*Client, error)

       // ListModels returns the names of all available models.
       func (c *Client) ListModels(ctx context.Context) ([]string, error)

       // Ping verifies the Ollama server is reachable by calling ListModels.
       func (c *Client) Ping(ctx context.Context) error
       ```

    1. Create `internal/chat/stream.go` implementing streaming chat:
       ```go
       package chat

       import (
           "context"
           "strings"
           "sync"

           "github.com/ollama/ollama/api"
       )

       // StreamChat sends the conversation to Ollama and streams the response.
       // onToken is called for each content chunk as it arrives.
       // The full assistant message is returned after streaming completes.
       // Context cancellation stops the stream (per D-04: Ctrl+C cancels active generation).
       func (c *Client) StreamChat(ctx context.Context, conv *Conversation, onToken func(string)) (*api.Message, error)
       ```
       Implementation details:
       - Build `api.ChatRequest{Model: conv.Model, Messages: conv.Messages}`.
       - Call `c.api.Chat(ctx, req, callback)` where callback:
         - Uses `sync.Once` to signal "first token arrived" (caller uses this to stop spinner).
         - Calls `onToken(resp.Message.Content)` for each non-empty content chunk.
         - Accumulates content in a `strings.Builder`.
         - Returns `ctx.Err()` to stop early if context is cancelled (per Pitfall 3 from RESEARCH.md).
       - After `Chat` returns, construct the full `api.Message{Role: "assistant", Content: fullContent.String()}`.
       - If the error is `context.Canceled`, return a recognizable error (not nil) so the caller knows streaming was cancelled vs failed.

    2. Add a `FirstTokenNotifier` helper:
       ```go
       // FirstTokenNotifier calls onFirst exactly once when the first token arrives.
       type FirstTokenNotifier struct {
           once    sync.Once
           onFirst func()
       }

       func NewFirstTokenNotifier(onFirst func()) *FirstTokenNotifier
       func (n *FirstTokenNotifier) Notify()
       ```
       This is used by the REPL to stop the spinner on first token.

    3. Create `internal/chat/stream_test.go` with:
       - To test without a live Ollama, refactor `StreamChat` to accept an interface or use a test helper that simulates the callback pattern.
       - Define a `mockChatFunc` that simulates `client.Chat()` by calling the callback with pre-defined responses.
       - `TestStreamChatAccumulatesTokens`: Simulate 3 token callbacks ("Hello", " ", "world"), verify returned message content is "Hello world".
       - `TestStreamChatCallsOnToken`: Verify `onToken` is called for each non-empty content chunk in order.
       - `TestStreamChatFirstTokenNotifier`: Verify `FirstTokenNotifier.Notify()` is called exactly once, on first token.
       - `TestStreamChatCancellation`: Cancel the context mid-stream, verify the error indicates cancellation.
       - Use `testify/assert` and `testify/require`.

    NOTE on testability: The Ollama `api.Client.Chat` method is not on an interface, making it hard to mock directly. The recommended approach is to extract the chat call into an internal interface:
    ```go
    // chatAPI is an internal interface for testing the streaming logic.
    type chatAPI interface {
        Chat(ctx context.Context, req *api.ChatRequest, fn api.ChatResponseFunc) error
    }
    ```
    Then `Client` holds a `chatAPI` field (set to `c.api` in production, mock in tests). This keeps the public API clean while enabling thorough unit tests.
