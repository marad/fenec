    **Step 1: Add SDK dependency**
    Run `go get github.com/openai/openai-go/v3@v3.31.0` to add the official OpenAI Go SDK.

    **Step 2: Create `internal/provider/openai/openai.go`**

    Create the full adapter file with the following structure. Use `sdkoai` as the import alias for `github.com/openai/openai-go/v3` (per research patterns).

    **Package and imports:**
    ```go
    package openai

    import (
        "context"
        "encoding/json"
        "fmt"
        "regexp"
        "strings"

        "github.com/marad/fenec/internal/model"
        "github.com/marad/fenec/internal/provider"
        sdkoai "github.com/openai/openai-go/v3"
        "github.com/openai/openai-go/v3/option"
        "github.com/openai/openai-go/v3/packages/ssestream"
        "github.com/openai/openai-go/v3/shared/pagination"
    )
    ```

    **Compile-time interface check:**
    ```go
    var _ provider.Provider = (*Provider)(nil)
    ```

    **Test interface (narrow chatAPI pattern from Ollama adapter):**
    Define a `completionsAPI` interface wrapping only the SDK methods used:
    - `New(ctx, params, ...option.RequestOption) (*sdkoai.ChatCompletion, error)` — for non-streaming
    - `NewStreaming(ctx, params, ...option.RequestOption) *ssestream.Stream[sdkoai.ChatCompletionChunk]` — for streaming

    Define a `modelsAPI` interface:
    - `ListAutoPaging(ctx, sdkoai.ModelListParams, ...option.RequestOption) *pagination.PageAutoPager[sdkoai.Model]` — for listing models

    **Provider struct:**
    ```go
    type Provider struct {
        completions completionsAPI
        models      modelsAPI
    }
    ```

    **Constructor `New(baseURL, apiKey string) (*Provider, error)`:**
    - Build `[]option.RequestOption` slice
    - If `baseURL != ""`, append `option.WithBaseURL(baseURL)`
    - If `apiKey != ""`, append `option.WithAPIKey(apiKey)`. Else append `option.WithAPIKey("not-needed")` — this prevents the SDK from reading OPENAI_API_KEY env var and erroring for local providers like LM Studio
    - Append `option.WithMaxRetries(2)` for resilience
    - Create client: `client := sdkoai.NewClient(opts...)`
    - Return `&Provider{completions: client.Chat.Completions, models: client.Models}`

    Also create `newWithAPI(completions completionsAPI, models modelsAPI) *Provider` for testing (same pattern as Ollama adapter).

    **Name() string:** Return `"openai"`.

    **ListModels(ctx) ([]string, error):**
    - Call `p.models.ListAutoPaging(ctx, sdkoai.ModelListParams{})` to get an auto-paging iterator
    - Iterate with `for pager.Next() { ... pager.Current() ... }` collecting model IDs into a `[]string`
    - Check `pager.Err()` after iteration
    - Return the model ID list

    **Ping(ctx) error:**
    - Call `p.ListModels(ctx)` — same pattern as Ollama adapter
    - If error, return `fmt.Errorf("cannot connect to OpenAI-compatible endpoint: %w", err)`
    - If no models, return `fmt.Errorf("no models available from OpenAI-compatible endpoint")`
    - Otherwise return nil

    **GetContextLength(ctx, modelName) (int, error):**
    - OpenAI `/v1/models` does NOT expose context length (per research Pitfall 3)
    - Return `0, nil` to signal "unknown/use model default" — the ContextLength field is only used by Ollama's `num_ctx` option, and the OpenAI API handles context limits server-side

    **StreamChat(ctx, req, onToken, onThinking) (*model.Message, *model.StreamMetrics, error):**
    This is the core method. Dispatch based on whether tools are present:

    If `len(req.Tools) > 0`: call `p.chatNonStreaming(ctx, req, onToken, onThinking)` (non-streaming path)
    If `len(req.Tools) == 0`: call `p.chatStreaming(ctx, req, onToken, onThinking)` (streaming path)

    **chatStreaming (private method):**
    - Build params via `buildParams(req)` helper
    - Call `p.completions.NewStreaming(ctx, params)`
    - `defer stream.Close()`
    - Iterate: `for stream.Next() { chunk := stream.Current(); ... }`
    - For each chunk: if `len(chunk.Choices) > 0`, get `delta := chunk.Choices[0].Delta`
    - If `delta.Content != ""`: append to content builder, call `onToken(delta.Content)` if non-nil
    - After iteration: check `stream.Err()`, return error if present
    - Build `model.Message{Role: "assistant", Content: content.String()}`
    - Call `extractThinking(&msg)` on the final content (for `<think>` tags in streamed content)
    - Return `&msg, &model.StreamMetrics{}, nil` (streaming metrics not available without stream_options)

    **chatNonStreaming (private method):**
    - Build params via `buildParams(req)` helper
    - Call `p.completions.New(ctx, params)`
    - If error, return wrapped: `fmt.Errorf("openai chat completion: %w", err)`
    - If `len(completion.Choices) == 0`, return error: `"openai: no choices in response"`
    - Get `choice := completion.Choices[0]`
    - Build `msg := model.Message{Role: "assistant", Content: choice.Message.Content}`
    - Parse tool calls from `choice.Message.ToolCalls`:
      ```go
      for _, tc := range choice.Message.ToolCalls {
          var args map[string]any
          if err := json.Unmarshal([]byte(tc.Function.Arguments), &args); err != nil {
              args = map[string]any{"_raw": tc.Function.Arguments}
          }
          msg.ToolCalls = append(msg.ToolCalls, model.ToolCall{
              ID: tc.ID,
              Function: model.ToolCallFunction{
                  Name:      tc.Function.Name,
                  Arguments: args,
              },
          })
      }
      ```
    - Call `extractThinking(&msg)` for reasoning_content and `<think>` tags
    - If non-streaming and content exists, call `onToken(msg.Content)` once (deliver full content to callback)
    - Map usage: `metrics.PromptEvalCount = int(completion.Usage.PromptTokens)`, `metrics.EvalCount = int(completion.Usage.CompletionTokens)`
    - Return `&msg, &metrics, nil`

    **buildParams(req *provider.ChatRequest) sdkoai.ChatCompletionNewParams:**
    - Set `Model: req.Model`
    - Set `Messages: toOpenAIMessages(req.Messages)`
    - If `len(req.Tools) > 0`: set `Tools: toOpenAITools(req.Tools)`
    - Return params

    **toOpenAIMessages(msgs []model.Message) []sdkoai.ChatCompletionMessageParamUnion:**
    Convert each message by role:
    - `"system"` -> `sdkoai.SystemMessage(m.Content)`
    - `"user"` -> `sdkoai.UserMessage(m.Content)`
    - `"assistant"` with tool calls -> Build `sdkoai.ChatCompletionAssistantMessageParam` with `ToolCalls` containing `ChatCompletionMessageToolCallParam` entries (marshal `tc.Function.Arguments` to JSON string for `Function.Arguments` field). Wrap in `sdkoai.ChatCompletionMessageParamUnion{OfAssistant: &param}`.
    - `"assistant"` without tool calls -> `sdkoai.AssistantMessage(m.Content)`
    - `"tool"` -> `sdkoai.ToolMessage(m.ToolCallID, m.Content)` — NOTE: the SDK's ToolMessage takes (toolCallID, content), verify argument order at implementation time
    - IMPORTANT: Do NOT include `Thinking` field in outgoing messages (per Pitfall 5 — DeepSeek returns 400)

    **toOpenAITools(tools []model.ToolDefinition) []sdkoai.ChatCompletionToolParam:**
    Convert each tool definition:
    - Build properties map: for each `name, prop` in `td.Function.Parameters.Properties`, create `map[string]any{"type": prop.Type[0]}` (PropertyType is `[]string`, use first element). Add `"description"` if non-empty. Add `"enum"` if non-empty.
    - Use `sdkoai.ChatCompletionToolParam` with `Function` field set to `sdkoai.FunctionDefinitionParam{Name: ..., Description: ..., Parameters: sdkoai.FunctionParameters{...}}`
    - Note: Verify exact SDK type names at implementation time — the research shows `ChatCompletionToolUnionParam` but v3 SDK may use `ChatCompletionToolParam`

    **extractThinking(msg *model.Message):**
    Two-stage extraction (per locked decision — opportunistic thinking):
    1. Check for `reasoning_content` in response extra fields — this is tricky since the non-streaming response's `choice.Message.JSON.ExtraFields` needs to be passed in. Consider passing the `choice` or the relevant extra fields map as a parameter instead of just the message. For streaming, skip this (no extra fields on chunks).
    2. If `msg.Thinking` is still empty after step 1, check content for `<think>...</think>` tags using `regexp.MustCompile("(?s)<think>(.*?)</think>")`. If found, extract to `msg.Thinking`, strip tags from `msg.Content` using `ReplaceAllString`.

    For the non-streaming path, extract reasoning_content like this:
    ```go
    if field, ok := choice.Message.JSON.ExtraFields["reasoning_content"]; ok {
        raw := field.Raw()
        var reasoning string
        if err := json.Unmarshal([]byte(raw), &reasoning); err == nil && reasoning != "" {
            msg.Thinking = reasoning
        }
    }
    ```

    **Regex variable** (package-level):
    ```go
    var thinkRegex = regexp.MustCompile(`(?s)<think>(.*?)</think>`)
    ```
