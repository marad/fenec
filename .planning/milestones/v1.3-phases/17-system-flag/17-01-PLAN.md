## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Information Disclosure | `os.ReadFile(*systemFile)` path traversal | accept | User explicitly provides the file path from their own shell session. They already have full shell access — no privilege boundary is crossed. Path traversal is the intended behavior (user can point to any file they can read). |
| T-17-02 | Tampering | Symlink following via `os.ReadFile` | accept | `os.ReadFile` follows symlinks, which is correct for user-provided paths. The user controls their own filesystem. No different from `cat <file>`. |
| T-17-03 | Denial of Service | Large file via `--system` | accept | User is running their own CLI tool locally. Reading a large file only affects their own session. No multi-user or network boundary. |
</threat_model>
